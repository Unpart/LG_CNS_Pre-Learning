1. JDK 설치와 환경 변수(Environment Variable)의 아키텍처적 의미

JDK (Java Development Kit) 17 채택: 실무 레거시(Java 11)와 최신 모던 프레임워크(Spring Boot 3.x 이상)의 요구사항을 모두 충족하며 LTS(Long Term Support)를 보장하는 버전이다. Eclipse 프로젝트에서 제공하는 오픈소스 기반의 Adoptium(Temurin) 배포판을 설치했다.

JAVA_HOME 설정의 당위성:

단순히 터미널에서 java 명령어를 치기 위해서라면 Path 변수에 bin 폴더만 등록해도 동작한다.

하지만 굳이 JAVA_HOME이라는 시스템 변수를 명시적으로 생성하는 이유는, 향후 사용하게 될 Spring Boot, Tomcat, Gradle, Maven 같은 엔터프라이즈 프레임워크 및 빌드 툴들이 시스템에서 JVM을 구동하기 위해 내부적으로 JAVA_HOME 경로를 참조(Dependency)하기 때문이다.

2. Java의 컴파일 파이프라인 (Compiler & Interpreter)

C/C++ 같은 언어는 컴파일러가 소스 코드를 특정 OS 종속적인 '바이너리 코드(Binary Code, 0과 1)'로 번역한다. 하지만 Java는 컴파일과 인터프리트의 하이브리드 방식을 취한다.

javac (컴파일): 인간이 작성한 고급 언어 소스 코드(.java)를 JVM이 이해할 수 있는 중간 언어인 바이트코드(Bytecode, .class)로 번역한다. (이 단계까진 OS가 코드를 이해할 수 없다.)

java (인터프리터 및 JIT): 바이트코드를 JVM 내부의 인터프리터가 읽어 런타임에 해당 OS가 이해할 수 있는 기계어로 해석 및 실행한다.

💡 핵심 아키텍처: 바이트코드는 OS에 종속적이지 않고 순수하게 JVM에만 종속적이다. 따라서 로컬 머신의 OS 종류에 상관없이 JVM만 설치되어 있다면 "Write Once, Run Anywhere"가 실현된다.

3. VS Code 통합 개발 환경(IDE) 세팅 및 Entry Point

순수 텍스트 에디터인 VS Code를 Java IDE처럼 활용하기 위해 Microsoft에서 제공하는 Extension Pack for Java를 설치하여 코드 자동 완성 및 디버깅 환경을 구축했다.

public static void main(String[] args):

Java 애플리케이션의 실행 진입점(Entry Point)이다.

클래스 템플릿(명사+동사 구조) 내부에 정의되며, JVM은 애플리케이션 실행 시 메모리에 이 main 메서드를 가장 먼저 로드하여 메인 스레드(Main Thread)의 작업 흐름을 시작한다.

네이밍 컨벤션 (Naming Convention):

Class: PascalCase (GreetingApp) - 첫 글자 대문자 필수 (파일명과 클래스명 일치 필수).

Method/Variable: camelCase (calculateSum) - 첫 글자 소문자, 이후 단어 조합 시 대문자.

Java는 철저하게 대소문자를 구별(Case-sensitive)하는 언어이므로 식별자(Identifier) 작성 시 컨벤션을 엄격히 지켜야 컴파일 에러를 방지할 수 있다.
